# Astra Linux: Подробный IR План

## Шаг 1. Фиксация среды
### Команда
```bash
script -a astra_ir.log
```
Что делает: записывает все действия в лог.
Параметры: `-a` дописывание.

### Команда
```bash
date -u && timedatectl && hostnamectl
```
Что делает: фиксирует время, таймзону и данные хоста.
Параметры: `-u` UTC.

## Шаг 2. Проверка мандатной модели
### Команда
```bash
astra-mic-control status
```
Что делает: статус MIC.
Параметры: `status` режим проверки.

### Команда
```bash
pdpl-user root
```
Что делает: проверка атрибутов root.
Параметры: имя пользователя.

### Команда
```bash
pdpl-file /etc/ssh
```
Что делает: проверка атрибутов критичного каталога.
Параметры: путь.

## Шаг 3. Классический Linux triage
### Команда
```bash
who; w; last -a | head -n 50
```
Что делает: входы и активность пользователей.
Параметры: `-a` IP/host у входов.

### Команда
```bash
ps aux --sort=-%cpu | head -n 40
```
Что делает: подозрительные процессы.
Параметры: `--sort=-%cpu` по CPU.

### Команда
```bash
ss -tulpen
```
Что делает: сеть/порты/процессы.
Параметры: `-t/-u/-l/-p/-e/-n`.

### Команда
```bash
find /tmp /var/tmp /dev/shm -type f -mmin -360 2>/dev/null
```
Что делает: следы недавнего запуска во временных папках.
Параметры: `-mmin -360` последние 6 часов.
